# logistic-gof-benchmark

Reproducibility materials for the paper **"Benchmarking Goodness-of-Fit and Calibration Algorithms
for Logistic Regression Classifiers: A Large-Scale Simulation Study under Sparse Data."**

[![DOI](https://zenodo.org/badge/DOI/10.5281/zenodo.XXXXXXX.svg)](https://doi.org/10.5281/zenodo.XXXXXXX)

> **DOI:** minted automatically by Zenodo when the first GitHub release is published (replace `XXXXXXX`
> above and in `CITATION.cff` / `.zenodo.json` once assigned).

This repository contains the simulation drivers, aggregated results, real-data example, and figure code
needed to reproduce every number and figure in the paper. It complements the R package
[`ebrahim.gof`](https://github.com/ebrahimkhaled/ebrahim.gof), which implements the tests.

## What the study does
A neutral, head-to-head simulation benchmark of ~25 goodness-of-fit and calibration tests for logistic
regression, across **5 covariate distributions × 5 sample sizes (200–5000) × 10,000 replications**, plus
four power scenarios (omitted quadratic / interaction, each slight and pronounced) and a low-birth-weight
real-data application.

## Reproducing

**Requirements:** R (>= 4.4), and the packages `ebrahim.gof`, `ResourceSelection`, `rms`, `givitiR`,
`BAGofT`, `mgcv`, `ggplot2`, `dplyr`, `scales`, `viridisLite`. Install the test package with
`remotes::install_github("ebrahimkhaled/ebrahim.gof")`.

1. **Figures only (fast):** run `R/make_heatmaps.R`, `R/03_excluded_tests_figure.R`, `R/make_reliability.R`
   against the CSVs in `data/` — no simulation needed.
2. **Full benchmark (slow, ~hours):** run `R/01_simulation_driver.R` to regenerate the per-replication
   `pvalues_*.csv`, then `R/02_summarize_and_heatmaps.R` to aggregate and plot.
3. **Real-data example:** run `R/04_realdata_lowbirthweight.R` on `data/lowbirthweight_*.csv`.
4. **EF / EDGE add-on:** run `R/05_ef_edge_grid_rerun.R` (validated against the HL anchor).

The full per-replication p-value matrices (`pvalues_*.csv`, ~60 files, ~180 MB) are **not** committed to keep
the repo light; they are archived in the Zenodo record and are regenerated by step 2.

## Seeds & environment
Runs use fixed seeds under R 4.4.1 with the default Mersenne-Twister generator on a 24-core machine; exact
package versions are pinned in the `ebrahim.gof` `DESCRIPTION`.
